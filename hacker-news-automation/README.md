# Hacker News Automation

## Project Overview

This is an **n8n workflow automation project** that fetches items from Hacker News based on a scheduled trigger. The workflow is designed to run automatically at specified intervals and retrieve Hacker News content filtered by keywords.

### Purpose

- Automatically fetch Hacker News items on a schedule
- Filter content by specific keywords (currently configured for "automation")
- Enable automated monitoring of Hacker News for topics of interest

### Technologies

- **n8n** - Workflow automation tool
- **Hacker News API** - Data source via n8n's built-in Hacker News node

## Project Structure

```
hacker-news-automation/
├── workflow.json    # n8n workflow definition
└── README.md          # Project documentation
```

## Workflow Details

The workflow (`workflow.json`) consists of two nodes:

| Node | Type | Description |
|------|------|-------------|
| **Schedule Trigger** | `n8n-nodes-base.scheduleTrigger` | Triggers the workflow daily at 9:00 AM |
| **Get many items** | `n8n-nodes-base.hackerNews` | Fetches Hacker News items with keyword filter "automation" |

### Connection Flow

```
Schedule Trigger → Get many items
```

## Building and Running

### Prerequisites

- [n8n](https://n8n.io/) installed and configured locally or on a server

### Running the Workflow

1. **Import the workflow** into n8n:
   - Open n8n editor
   - Click "Import from File" and select `workflow.json`

2. **Activate the workflow**:
   - Toggle the workflow status to "Active" in the n8n UI
   - The workflow will trigger automatically at 9:00 AM daily

3. **Test manually**:
   - Click "Execute Workflow" in the n8n editor to test immediately

### Configuration

To modify the workflow:

- **Schedule**: Edit the `triggerAtHour` value in the Schedule Trigger node
- **Keyword**: Change the `keyword` field in the "Get many items" node parameters

## Development Conventions

- Workflow files follow n8n's JSON export format
- Each workflow should be self-contained in its own directory
- Workflow IDs are auto-generated by n8n and should be preserved
